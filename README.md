# Autonomous Inventory & Markdown Agent

A production-grade, AI-powered multi-agent retail decision system. This platform autonomously analyzes inventory health, sales history, and profitability data to dynamically recommend optimal markdowns and pricing actions. It demonstrates a complete LangGraph workflow orchestrated alongside FastAPI, React 19, and the Model Context Protocol (MCP).

## What This Project Does

The Autonomous Inventory & Markdown Agent (AIM) acts as an intelligent retail assistant. By leveraging a multi-agent AI architecture (powered by LangGraph), it autonomously:
1. **Analyzes Inventory**: Checks current stock levels and identifies slow-moving or overstocked products.
2. **Evaluates Demand & Sales**: Looks at historical sales velocities to forecast future demand.
3. **Calculates Optimal Pricing**: Suggests intelligent markdowns to clear excess stock without unnecessarily sacrificing profit margins.
4. **Assesses Risk**: Enforces strict business rules to prevent extreme markdowns that could result in losses.
5. **Generates Executive Reports**: Compiles data-driven recommendations into actionable reports for human review.

This system effectively bridges the gap between raw inventory data and strategic retail pricing decisions.

## Application Pages & Navigation

The frontend React dashboard is divided into several dedicated pages to manage the full lifecycle of inventory analysis:

- **Dashboard (`/`)**: A real-time command center providing high-level KPIs (Total Products, Out of Stock, Active Agents) and a feed of recent analyses and agent activities.
- **Upload (`/upload`)**: A drag-and-drop interface allowing users to import new product catalogs and inventory data via CSV or JSON files.
- **Inventory (`/inventory-list`)**: A comprehensive table view of all tracked products. Users can search, view, edit, or delete items, monitoring their stock quantities, unit costs, and current prices.
- **Analysis (`/analyze`)**: The core AI execution hub. Users can trigger the LangGraph multi-agent pipeline and watch a live stream of logs as the agents process data in real-time.
- **Monitor (`/monitor`)**: Provides a detailed, real-time look into the status of each individual AI agent (Inventory, Sales, SQL, Pricing, Risk, Report) and tracks their historical MCP (Model Context Protocol) tool calls.
- **Reports (`/reports`)**: Displays the finalized executive markdown recommendations generated by the AI. Users can review the suggested prices, risk levels, and approve/reject actions, as well as export the data to CSV.
- **Settings (`/settings`)**: A configuration panel to adjust global agent behaviors, such as setting the maximum allowed markdown percentage and default risk thresholds.
- **Help (`/help`)**: In-app documentation and step-by-step guides on how to utilize the AIM platform effectively.

## Features

- **Multi-Agent Orchestration**: A custom LangGraph workflow connecting 6 distinct agents to compute data-driven decisions.
- **Model Context Protocol (MCP)**: Implementation of dedicated MCP servers (SQL, Inventory, Pricing) that provide specialized tool access to the agents.
- **Real-Time Streaming**: Server-Sent Events (SSE) stream live agent execution statuses and workflow progressions directly to the frontend.
- **Retail Operations Dashboard**: A React 19 + Vite frontend providing KPI tracking, file uploads, live execution monitoring, and markdown action reports.
- **Data Persistence**: Robust PostgreSQL integration utilizing SQLAlchemy 2.0 ORM and Alembic for automated schema migrations.
- **Containerized Infrastructure**: Fully containerized using Docker and Docker Compose for zero-config deployments.

## Steps to Run

1. **Clone the Repository**
   Navigate to your workspace and clone/extract the project files.

2. **Configure Environment Variables**
   Ensure the `.env` file exists in the root directory and populate your API keys:
   ```env
   POSTGRES_USER=admin
   POSTGRES_PASSWORD=admin
   POSTGRES_DB=inventory_db
   GEMINI_API_KEY=your_api_key_here
   ```

3. **Deploy using Docker Compose**
   Launch the entire stack (PostgreSQL, FastAPI Backend, React Frontend) via Docker Compose:
   ```bash
   docker-compose up --build
   ```

4. **Access the Application**
   - **Frontend UI:** Open `http://localhost:5173` in your browser.
   - **Backend API Docs:** Open `http://localhost:8000/docs` to view the auto-generated Swagger UI.

5. **Run Migrations (If running locally without Docker)**
   ```bash
   cd backend
   python -m venv venv
   source venv/bin/activate  # Or .\venv\Scripts\Activate.ps1 on Windows
   pip install -r requirements.txt
   alembic upgrade head
   ```
